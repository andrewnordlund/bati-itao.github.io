Web Accessibility Checklist

Level 1 - Basic

  1. I can use Your App With A Keyboard Only (ie: without the need for a mouse or touch)?
    • I can Tab through all pages top to bottom, and Shift+Tab backwards from bottom to top.
      • ...in a logical order
      • I can always see my focus indicator
      • My focus doesn't land on non-interactive, non-scrollable elements
    • All tasks can be accomplished with the keyboard

Level 2 - Intermediate

  1. When I zoom into 200% everything works, and nothing is lost.
  2. When I view at 320px wide, everything works (including with a keyboard), nothing is lost, and the page only scrolls up and down.
  3. All pages have a unique and descriptive <title>
  4. All <h#> heading text is descriptive
  5. <h#> heading levels aren't skipped.
  6. The purpose of each link can be determined from the link text. (i.e.: not "Click here", or "Read more".)

Level 3 - Advanced

  1. ID values are unique on each page.
  2. All pages are "Axe-clean" (i.e. the Axe DevTools browser extension gives no errors).
  3. There is a 4.5:1 contrast ratio between text and its background.
  4. All <table> data cells (<td> elements) are programmatically associated to appropriate <th> cells.
  5. All inputs have a visual indication of what they're for.
  6. All inputs have a programmatically determinable accessible name (through associated <label>, aria-label, or aria-labelledby).
