Frequently Asked Questions - Web Accessibility

  • Do we need to submit a request each time we make a change to our applications?

    An accessibility audit is required when the user interface has been changed.

  • What are current and future accessibility standards?

    Current standards

    • WCAG 2.1 (A and AA) supported by
      • Guideline on Making Information Technology Usable by All
      • Standard on Web Accessibility (currently being refreshed)
      • The European Standard for Digital Accessibility (EU EN 301 549 v2.1.2)

    Future standards

    • WCAG 2.2 (A and AA) (end of 2021)
      • Will be supported by the latest update to the Standard on Web Accessibility or one of the upcoming digital accessibility policies.
    • WCAG 3.0 (silver rating)(estimated towards 2025)
      • Unknown yet how this will be supported but most likely through an updated TBS policy instrument
      • WCAG 3.0 will be a complete shift of how conformance testing is performed.
  • How to validate internal web pages against the Nu Html Checker?

    To validate an internal page, one can use the “Check serialized DOM for the current page” bookmarklet and the “WCAG Parsing” Bookmarklet to filter the result. To try out the bookmarklets, please follow the steps below:

    1. Save the Check serialized DOM of current page
    2. Save the WCAG parsing only as a bookmark.
    3. Check a page (test page) using the Check serialized DOM of current page bookmarklet)
    4. Once the results page is displayed activate the Parsing bookmarklet
    5. The results will be filtered to provide a rough subset of the validation results that represent WCAG 2.x 4.1.1 Parsing fails.

  • How to code email addresses?

    Generally, it is recommended to provide the full email address as link text. For example: “Please contact the Accessible Information and Communication Technology for more information: edsc.ti-it.a11y.esdc@hrsdc-rhdcc.gc.ca“ or “Please email edsc.ti-it.a11y.esdc@hrsdc-rhdcc.gc.ca“

    Please write the email address in lower case as recommended by the Canada.ca Content Style Guide.

  • Could you please provide us with the defect severity scale used in the assessment of internal staff COTS products?

    We don’t have a defect severity scale or at least not an official one. Remember that we don’t want clients saying “We only missed one success criteria and got 99% so we’re good to go”. That one missed success criteria could be keyboard accessibility and would have a huge impact versus a site that got 85% but all low impact issues. At the end of the day our role is to identify and guide on accessibility.
